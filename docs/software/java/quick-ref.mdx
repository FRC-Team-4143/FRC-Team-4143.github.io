# Quick Reference Guide
This is a quick reference guide for Java programming. It provides an overview of the key concepts throughout this section. It is not meant to be a comprehensive guide, but rather a quick reference for the most common concepts and syntax used in Java programming.

- Use comparison operators inside `()` for if-statements.
- Only compare compatible types.
- Example: `if (x > 0) { ... }`
## [**Loops**](./loops.mdx)
- Use `for` and `while` loops to repeat code.
- Avoid `while` loops in robot code to prevent infinite loops.
- Use increment operators (`++`, `--`, `+=`, `-=`) for counters.
## [**Arrays & ArrayLists**](./arrays.mdx)
- Arrays store multiple values of the same type. Indexing starts at 0.
- Use `.length` for array size and `ArrayList.size()` for ArrayLists.
- Arrays are fixed size and ArrayLists are dynamic arrays.
## [**Classes & Objects**](./classes-objs.mdx)
- Java is class-based. Classes are blueprints; objects are instances.
- Use `PascalCase` for class names.
- Use constructors to initialize objects.
- Access fields and methods with dot notation.
- Prefer getter/setter methods for encapsulation.
## [**Methods**](./methods.mdx)
- Methods are reusable blocks of code.
- Use `camelCase` for method names.
- Methods can have parameters and return values.
